# State: schema-migration

## Goal

Add `stop_reason TEXT` (nullable, default NULL) and `max_tokens INTEGER` (nullable, default NULL) columns to `taskUsageTable` in `schema.ts`. Generate the drizzle migration file.

## Semantic distillation

Two nullable columns added to the existing table. No FK constraints. No index needed — `stop_reason` is not a query filter. `max_tokens` is a constant per task (written once at task-start). `stop_reason` updates per-response but the final value is stored (most-severe wins). Both columns are nullable so old rows (pre-migration) contain NULL, which is the correct "unknown" default.

## Notes

- Column names use snake_case in the DB (Drizzle convention) and camelCase in TypeScript (`stopReason`, `maxTokens`). The Drizzle column definition maps between them automatically.
- Do NOT add a `.notNull()` constraint — both columns must be nullable. Old rows will have NULL and that is the correct default.
- If `npx drizzle-kit generate` prompts interactively, run it with `--custom` flag or just accept the default migration name.
- The Drizzle `$inferSelect` type for `taskUsageTable` will automatically include the new nullable columns as `string | null` and `number | null` respectively. `tools/usage.ts` uses `TaskUsageRow = typeof taskUsageTable.$inferSelect` — this picks up the change without any edits to `usage.ts` in this state.
