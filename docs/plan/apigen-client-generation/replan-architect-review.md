# Architect Review — apigen v2 Replan

**Reviewer:** architect-reviewer (read-only)
**Date:** 2026-06-22
**Scope:** the replanned `plan-state-machine` tail (v2 phases) in `docs/plan/apigen-client-generation/`, judged against `docs/apigen/SPEC.md` (v2 canonical standard).
**Inputs read:** `SPEC.md` (§0–§16), `orchestration-ledger.md` (REPLAN + F30–F34), `dag.json` (all 768 lines), `state.json` (provenance + all v2 state entries), `README.md` (Value delta + DoD dod.1–13), `scripts/audit_apigen.py` (phase map + `phase_final` stub bindings, lines 487–977), `scripts/probe_mcp.mjs` (entrypoint + body-envelope), and 19 `contexts/*.md`. Repo state cross-checked with `find packages/apigen -name project.json`.

---

## Verdict: **needs-rework**

The replan's *intent* is sound and faithful: the REPLAN section of the ledger and the dag node `notes` map cleanly onto SPEC §4–§14, the DAG topology gates `done` correctly behind all v2 work, and F28/F29 are placed at the right layer (`ts-extractor-by-symbol`, by exported symbol). But the plan **cannot be executed as authored** and **cannot prove what it claims**, for four independent, individually-blocking reasons:

1. **Three v2 audit guards are dead on arrival** — they invoke audit phases (`v2-core`, `v2-harness`, `v2-projection`) that `audit_apigen.py` does not implement and that its `argparse(choices=…)` will *reject* with exit 2. Every v2 audit gate fails for a missing-phase reason, not a product reason (BLOCKER-1).
2. **All 17 v2 states carry zero acceptance criteria and empty Goals** — the contexts are bare scaffolds (`_No criteria yet._`, `<What is true after…>`). The skill's "criterion → mirrored audit check ID" contract is unsatisfied for the entire v2 surface, so no v2 work state's guard actually proves the state's intent (BLOCKER-2).
3. **Nine new v2 packages are never scaffolded** — `apigen-naming/errors/schema/conformance/gateway`, `codegen/openapi`, and the `logger/openapi/health` plugins are referenced by guards and reservations as if they exist, but no state creates them, and none of the F13/F16 nx wiring (tsconfig.base path, vite.config, tsconfig.lib/spec) that bit v1 five times is reserved (BLOCKER-3).
4. **The re-pointed v1 behavioral DoD contradicts the v2 architecture it is supposed to guard** — dod.1/2/5 assert the `{data:…}` *body* envelope and drive the `npx tsx …/src/index.ts` entrypoint that F25/F26 already proved broken; both are flatly inconsistent with §9 (metadata envelope) and the F26 bundled-bin fix (BLOCKER-4).

These are fixable with targeted edits (below). The architecture underneath is the right one.

## Sequencing & risk notes

**Recommended ordering correction (R3 + R6 combined):**
```
canonical-descriptor → [v2-scaffold: create naming/errors/schema/conformance/gateway/codegen-openapi/plugin-{logger,openapi,health} IN FINAL §12 HOMES]
  → naming-helpers, error-taxonomy, plugin-interface (parallel, deps satisfied)
  → ts-extractor-by-symbol, layer-harness (parallel)
  → central-validation, projection-transports, logger-layer-plugin, mount-plugins
  → conformance-vectors  →  (verification-only) package-restructure
  → unified-cli → gateway
  → integration-tests-v2 → audit-final-v2 → done
```
Scaffolding in final homes turns `package-restructure` from a churning rename (R6) into a cheap layout-verification gate, and removes BLOCKER-3's missing-project failures.

**Biggest architectural risks, ranked:**
1. **R1 (dead audit phases)** — the v2 tail is literally unreachable today; nothing downstream can be proven. Fix first.
2. **R2 (zero v2 criteria)** — even after R1, every v2 work-state guard is a proxy ("tests pass") with no teeth tied to the state's intent. This is the v1 F3 failure re-introduced wholesale; the replan green-lit `gap-check PASSED` because gap-check validates the dod.9–13 clauses in README + `phase_final`, not the empty per-state criteria.
3. **R4 (v1 DoD contradicts v2)** — the kept regression guards assert a deleted contract; they will either fail at audit-final or, worse, pass and pin the wrong behavior, fighting dod.11.
4. **R9 (gateway unprovable single-host)** — dod.12 risks a scripted-stub pass that the real topology can't make (CLAUDE.md §6.5).
5. **R8 (silent §10/§11/§12-codegen scope drop)** — `done` will overclaim "v2 met."

**Guards that won't actually prove their criteria:** `audit-v2-core/harness/projection` (R1 — exit 2, prove nothing); every v2 work state (R2 — empty criteria → guard proves package-builds/tests-pass only); `integration-tests-v2` (R12 — whole-suite green ≠ the three v2 assertions exist with teeth); dod.6 (R7 — counts the wrong nine).

**What the replan got right (keep):** DAG terminal gating (`done → audit-final-v2 → integration-tests-v2 → all v2 frontier`) is correct and complete — `done` cannot fire until every v2 leaf is consumed. The 18 complete v1 states are correctly frozen as the "first TS host" (§15). F28/F29 layer placement is right (R13). The dod.9–13 → `phase_final` stub-binding *forcing function* (the "fail-until-built" stubs, lines 864–922) is a genuinely good pattern — it's just stranded behind R1 for the per-phase audits and unmirrored by per-state criteria (R2). The write-conflict instincts (GATE-C) are correct; they just need to be real DAG edges (R5).

**One-line bottom line for the planner amendment:** fix the audit-phase registry (R1), author real per-state v2 criteria (R2), add the missing-package scaffold phase in final §12 homes (R3+R6), and reconcile the kept v1 DoD with §9/§14 + the F26 bin (R4) — then the architecture (which is correct) becomes executable and provable.
