## F-P6-13 — agent-mcp@2.0.0 runtime deps not resolvable (live test, RELEASE-blocking)

The reload literally "reconnect failed" on this. Worked around locally by symlinking the 7 runtime-closure dist packages into `node_modules/@adhd/`. **Real fix (Plan 9 publish):** the `"*"` deps must become real published versions and resolve from the package; local dev needs a proper workspace install/link. Caught only by running the real server — the suite (vitest tsconfig-paths to source) never exercised dist resolution.

## F-P6-14 — migration 0006 silently skipped on existing DBs (live test, RELEASE-blocking, FIXED)
`0006_composed_prompts_cache`'s `_journal.json` `when` was `1750809600000` (≈Jun 2025) — OLDER than every prior migration — so drizzle's `when`-ordered migrator treats it as already-applied and SKIPS it on any DB past 0005. Fresh installs migrate fine; **every existing 1.x DB silently fails to upgrade** → new code (expecting `sessions.composed_prompt_id`) breaks. The green suite missed it (fresh DBs only). **Fixed** (`db53687`): corrected `when` to `1781497706486`; verified the upgrade now applies to a copy of the real 46-agent store (column + tables added, data preserved) and fresh DBs still work. This makes dod.3 ("nothing breaks") genuinely true for UPGRADES, not just fresh.
